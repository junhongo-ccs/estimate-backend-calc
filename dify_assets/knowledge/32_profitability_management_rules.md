# 収益性管理ルール (Profitability Management Rules)

## 1. 労務費単価 (Standard Labor Costs)
開発原価計算においては、以下の職位別月単価を使用すること。

| Rank | 月額単価 (円) | 備考 |
| :--- | :--- | :--- |
| **Rank 1** | 512,000 | 初級エンジニア相当 |
| **Rank 2** | 732,800 | 中級エンジニア相当 |
| **Rank 3** | 889,600 | 上級エンジニア・PM相当 |
| **Rank 4** | 1,088,000 | エキスパート・コンサルタント相当 |

### 標準チーム構成
見積もりの際は、以下の比率で加重平均した単価を使用する。
*   **Rank 2**: 50%
*   **Rank 3**: 50%
*   **平均月単価**: 約811,200円

## 2. 間接費 (Indirect Costs)
*   **間接費率**: 直接労務費の **38%**
*   計算式: `直接労務費 × 0.38`
*   内容: オフィス賃料、光熱費、福利厚生費、共通部門（人事・総務）の配賦など。

## 3. 販管費 (SG&A Results)
事業部基準および全社基準の販管費を、売上高に対して以下の料率で計上する。

*   **事業部販管費**: 売上の **18.6%**
*   **全社販管費**: 売上の **9.5%**
*   **合計販管費率**: 売上の **28.1%**

## 4. 利益の定義
*   **売上総利益 (Gross Profit)** = 売上高 - (直接労務費 + 間接費 + 外注費)
*   **営業利益 (Operating Profit)** = 売上総利益 - 販管費合計

## 5. リスクプレミアム (Risk Premium)
難易度の高いプロジェクト（モバイルアプリ開発、短納期案件など）においては、標準の利益率に加え、以下のリスクプレミアム（予備費）を確保することが推奨される。

*   **対象**: モバイルアプリ開発、短納期案件、新規技術採用案件
*   **考え方**: 原価（労務費）を増やすのではなく、**売価（見積額）に係数を乗じてバッファを確保** する。
*   **目的**: 手戻りやトラブル発生時のコスト吸収用。高利益率に見えるが、これは「利益」ではなく「リスク対策費」である。
